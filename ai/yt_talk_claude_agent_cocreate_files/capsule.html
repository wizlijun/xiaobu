注：部分语音识别错误已根据上下文进行修正，例如将 "Cloud Code" 修正为更准确的 "Claude Code"，后者是 Anthropic 旗下一款专注于编程的 AI 助手。 另外，访谈中提到的 "Cloud Developer Platform" 已被更名为 "Claude Developer Platform"。

平台演进：从 API 到综合开发者平台
  • 事实平台已从最初的 Anthropic API 更名为 Claude 开发者平台。
  • 事实新平台是一个综合体，包含 API、SDK、文档 及控制台体验，旨在为开发者提供构建所需的一切。
  • 观点更名反映了平台的巨大演变，在过去一年中增加了如 Prompt 缓存、批量 API、网络搜索、代码执行 等大量新功能。
  • 事实该平台不仅服务于外部客户，也支撑着 Anthropic 的内部产品，例如 Claude Code 就是直接构建在该公共平台之上的。
核心议题：到底什么是 AI Agent？
  • 观点"Agent" 一词目前有些被泛化，但 Anthropic 的核心定义是：模型能自主选择和调用工具，处理结果，并决定下一步行动的系统。
  • 事实Agent 与预定义路径的工作流 (Workflow) 不同，它更加灵活，能随着模型智能水平的提升而自动变得更强大。
  • 观点行业趋势正在从为模型设置过多“脚手架” (scaffolding) 的重型框架，转向更轻量、让模型发挥其内在推理能力的“解放模型” (un-hobble the model) 模式。
  • 观点开发者应该更多地提供辅助工具，而不是限制模型的“保险杠”，让模型自己去发现和创造解决方案。
开发者最佳实践与入门建议
  • 事实对于入门开发者，官方首要推荐使用 Claude Agent SDK（前身为 Claude Code SDK）。
  • 观点这是一个常见的误解：虽然名为 Code SDK，但它本质上是一个通用的 Agentic Harness（智能体框架），移除了编码特定部分后，可用于任何需要 Agent 循环和工具调用的场景。
  • 事实SDK 抽象并封装了管理工具调用循环和Prompt 缓存等通用逻辑，让开发者可以从一个更高的起点开始构建。
  • 观点企业在应用 Agent 时，应首先思考其商业价值，明确要解决的实际问题（如节省工时、减少手动工作），这有助于定义 Agent 的范围。
平台的先进工具与特性
  • 事实平台提供了服务端网络搜索 (Web Search) 和网页抓取 (Web Fetch) 工具，能让 Claude 自主进行深度研究。
  • 事实为解决长对话中的上下文窗口限制问题（即使有100万token），平台推出了新功能来帮助开发者管理上下文。
  • 事实一项关键特性是，模型可以智能地移除不再需要的旧工具调用记录，为更重要的信息腾出空间，同时会留下一个“墓碑”(Tombstone) 式的标记，告知模型这里曾有信息。
  • 事实平台引入了“记忆工具”(Memory Tool)，允许模型将关键信息和学习到的经验记录下来，并在未来的任务中参考，从而实现跨任务的成长和优化。 开发者可以控制这些记忆的存储位置。
未来展望：更智能、更易用的 Agent 未来
  • 观点未来的重点是提供更高层次的抽象，让开发者更容易地从 Claude 中获得最佳结果，并结合强大的可观察性 (Observability) 工具，以监控和引导长时间运行的任务。
  • 观点终极目标是创建一个“飞轮效应”，通过结合高级抽象、可观察性和记忆等功能，让用户构建的 Agent 能够自我改进、持续学习。
  • 事实Brad 表示，他最期待的是新模型的发布，因为每次发布都像打开新的可能性。
  • 观点另一个激动人心的方向是“给 Claude 一台计算机”。目前的代码执行工具只是第一步，未来希望 Claude 能拥有一个持久化的计算环境，可以像人类一样组织文件、配置工具，从而彻底“解放”其能力。
