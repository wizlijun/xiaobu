2025 Stack Overflow 开发者调查：AI 时代的技术栈与开发者画像

Stack Overflow 发布了第 15 年的年度开发者调查报告。这次调查覆盖了 177 个国家和地区的 49,000 多名开发者，涉及 62 个问题和 314 种技术。与往年不同的是，今年新增了 AI Agent、LLM、AI 工具和社区平台等重点领域，试图描绘出 AI 浪潮下开发者群体的真实状态。

这篇文章将从开发者画像、技术使用现状、AI 应用、职场状况等维度，梳理调查中的关键数据与趋势。

一、总体概况

本次调查的基本信息：

  • 参与人数：49,000+ 名开发者
  • 覆盖范围：177 个国家 / 地区
  • 调查深度：62 个问题，314 种技术
  • 连续年份：第 15 年
  • 新增重点：AI Agent、LLM、AI 工具、社区平台

二、开发者画像

国家分布

受访者的地理分布高度集中在欧美及印度：

  1. 美国：20.4%
  2. 德国：8.6%
  3. 印度：7.2%
  4. 英国：5.8%
  5. 法国：4%

编程经验

开发者的经验水平呈现明显的两极分布：

  • 65% 的开发者编码 10 年以上
  • 35% 编码少于 10 年

这说明行业仍在持续吸纳新血，同时也保留着大量资深从业者。

角色分布

今年的角色统计中，架构师作为新增角色首次出现：

  • 全栈开发：27%
  • 后端开发：14.2%
  • 学生：11.3%
  • 架构师（今年新增）：6.1%（第 4 大角色）

三、技术使用现状

编程语言

最常用的编程语言排名：

  1. JavaScript：66%
  2. HTML/CSS：52.2%
  3. Python：51%
  4. SQL：49.3%
  5. TypeScript：39.5%

最受喜爱的语言则呈现出不同的面貌：

  • Rust：79.5%（连续 5 年第一）
  • Elixir、Zig、Go、TypeScript 紧随其后

洞察：Rust 持续占据"最受喜爱"榜首，但使用率仅 13.2%。这反映出开发者对性能、安全、并发能力的向往，但现实生产环境中 Python 和 JavaScript 仍是主力。

数据库

数据库领域出现了明显的增长趋势：

  • PostgreSQL：55.6%（最常用 + 最受喜爱 + 最想学）
  • MySQL：40.5%
  • SQLite：37.5%
  • SQL Server：30.1%
  • Redis：28%（同比 +8%，缓存与实时需求驱动）

洞察：PostgreSQL 同时占据三个榜单的头部位置，已成为行业默认选择。

云与基础设施

容器化技术的增长尤为显著：

  • Docker：71.1%（同比 +17%，年度最大增幅）
  • AWS：43.3%
  • Kubernetes：28.5%

洞察：容器化已成为"默认技能"，Docker 在基础设施领域的地位类似于 JavaScript 在前端的地位。

Web 框架

主流框架保持稳定，新兴框架快速增长：

  • Node.js、React 持续领先
  • FastAPI 同比 +5%（年度最大增长之一）
  • Phoenix 使用率低但满意度最高（79%）

洞察：Python API 框架（FastAPI）快速崛起，"小众但高幸福感"的框架越来越受到关注。

IDE / 编辑器

开发工具市场格局清晰：

  • VS Code：75.9%（连续 5 年第一）
  • Visual Studio、IntelliJ、Vim 依旧稳固
  • AI IDE（Cursor、Claude Code）增长中，但未撼动 VS Code

洞察：插件式 AI 优于全 AI IDE，开发者更信任熟悉的工具 + 可选 AI。

四、最受关注的新兴技术

开发者对新技术的关注点集中在 AI 相关领域：

  • Large Language Model
  • Google Gemini
  • Ollama
  • RAG、c++23、Shadcn/ui、Pydantic 持续升温

五、AI 与 LLM

AI 工具使用率

AI 在开发者群体中的渗透率已经很高：

  • 84% 的开发者已使用或计划使用 AI
  • 47% 每天使用 AI
  • 专业开发者中 51% 每日使用

大模型使用

大模型的使用呈现"一超多强"格局：

  • OpenAI GPT：81.4%
  • Claude Sonnet：42.8%
  • Gemini Flash：35.3%

洞察：GPT 是"生态中心"，Claude Sonnet 是最受喜爱的 LLM，开发者倾向多模型并用。

AI Agent

AI Agent 的采用率远低于通用 AI 工具：

  • 每日使用：14%
  • 不使用 & 不打算用：38%

洞察：Agent 尚未主流，当前 AI 更多是"副驾驶"，而非"代理人"。

AI 信任度

开发者对 AI 的信任度仍然有限：

  • 信任 AI：33%
  • 不信任 AI：46%
  • 高度信任：仅 3%

洞察："几乎正确"是最大痛点，AI 需要人类验证，尤其在高责任岗位。

AI 的真实价值

开发者认为 AI 的主要价值在于：

  • 提升个人效率：69%
  • 减少重复劳动
  • 加快学习速度

六、职场与薪资

薪资水平

全球开发者薪资中位数及高薪角色：

  • 全球中位数：约 $60k
  • 高管：$139k
  • 工程经理：$130k
  • 金融 / 云 / 架构：$100k+

地区差异显著：

  • 美国工程经理：$200k
  • 印度工程经理：$52k

洞察：管理岗不等于技术深度，但仍是最强变现路径。

工作满意度

开发者的工作满意度有所提升：

  • 开心：24.5%（同比上升）
  • 躺平：47.1%
  • 不开心：28.4%

最重要的满意因素：

  1. 自主权
  2. 薪资
  3. 解决真实问题

AI 是否威胁工作？

大多数开发者对 AI 的态度相对乐观：

  • 不认为是威胁：63.6%（下降）
  • 认为是威胁：15%

信心有所下降，但恐慌尚未形成。

七、技术决策与工具偏好

影响技术采购

开发者在技术选型中的话语权较大：

  • 48% 的开发者影响或决定技术选型
  • 其中 20% 决定"核心技术栈"

喜欢技术的原因

开发者选择技术时最看重的因素：

  1. 好用的 API
  2. 完整的 API
  3. 质量与可靠性
  4. 成本可控
  5. 开源背景

值得注意的是，AI 集成排名倒数第 2。

八、核心洞察

综合整份调查，可以得出以下几点结论：

技术演进

  • Rust 是工程师的"精神信仰"，Python 是生产力现实
  • PostgreSQL + Docker + VS Code 构成新的技术栈共识层
  • 容器化已从"流行技术"变为"默认假设"

AI 现状

  • AI 已普及，但信任不足
  • Agent 潜力巨大，但尚未爆发
  • 开发者更看重基本功能，而非 AI 集成

开发者群体

  • 全栈 + 后端占据主流（41%）
  • 架构师角色正式成为显性主流
  • 持续学习是常态（69% 在过去一年学习了新技术）

社区价值

  • Stack Overflow 在 AI 时代反而更重要
  • 技术文档、在线资源、社区仍是学习的核心渠道
  • "用 AI 学 AI"成为新的学习路径（52.5%）

这份调查展现了一个处于转型期的开发者生态：AI 工具已经渗透到日常工作中，但开发者对其能力边界有清醒认知；传统技术栈依然稳固，但新技术在特定领域快速增长；开发者群体的经验分布呈两极化，但持续学习的意愿普遍存在。技术选型的核心依然是可靠性、API 质量和成本，而非炫技或追逐热点。
