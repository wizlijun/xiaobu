GitHub Octoverse 2025：每秒一位新开发者加入，AI 助推 TypeScript 登顶

GitHub 发布了 2025 年度 Octoverse 报告，揭示了 AI、Agent 和强类型语言如何驱动十多年来软件开发领域最大的变革。本文是对该报告的完整翻译，包含所有核心数据和趋势洞察。

如果要给 2025 年定一个主题，那就是"增长"。平均每秒就有超过一位新开发者加入 GitHub，过去一年新增超过 3600 万用户，这是 GitHub 有史以来最快的绝对增长速度，目前已有超过 1.8 亿开发者在 GitHub 上工作和构建项目。

2025 年度核心数据速览

  • 630M：GitHub 总项目数
  • 180M+：开发者总数
  • 1.12B：总贡献数
  • 4.3M：AI 项目数
  • 43.2M：每月平均合并的 Pull Request 数（同比增长 23%）
  • TypeScript 和 Python：2025 年使用最多的两门语言

2024 年底 GitHub Copilot Free 的发布与开发者注册量的阶梯式增长同时出现，超出了此前的预测。除了为生态系统带来数百万新开发者外，仓库、Pull Request 和代码推送的活动量也创下历史新高。开发者每分钟创建更多新仓库，每月平均合并 4320 万个 Pull Request（同比增长 23%），2025 年推送近 10 亿次 commit（同比增长 25.1%），其中仅 8 月就创下近 1 亿次的记录。

结构性里程碑：2025 年 8 月，TypeScript 首次超越 Python 和 JavaScript，成为 GitHub 上使用最多的语言，反映出开发者正在重塑他们的技术栈。这是十多年来最重大的语言格局变化。

增长是全球性的：仅印度今年就新增超过 500 万开发者（占所有新账户的 14% 以上），预计到 2030 年，每三位 GitHub 新开发者中就有一位来自印度。

今年数据揭示的三大关键转变

1. 生成式 AI 已成为开发标配

超过 110 万个公开仓库现在使用 LLM SDK，其中 693,867 个项目是在过去 12 个月内创建的（同比增长 178%，2025 年 8 月 vs 2024 年 8 月）。开发者还创下合并 5.187 亿 Pull Request 的记录（同比增长 29%）。此外，AI 采用从一开始就很快：80% 的 GitHub 新开发者在第一周就使用 Copilot。

2. TypeScript 现在是 GitHub 上使用最多的语言

2025 年 8 月，TypeScript 超越了 Python 和 JavaScript。它的崛起说明开发者正在转向强类型语言，使 Agent 辅助编码在生产环境中更加可靠。几乎所有主流前端框架现在默认使用 TypeScript 来搭建项目脚手架。尽管如此，Python 在 AI 和数据科学工作负载中仍然占主导地位，而 JavaScript/TypeScript 生态系统的整体活动量仍然超过 Python。

3. AI 正在重塑选择，而不仅仅是代码

过去，开发者的选择意味着挑选 IDE、语言或框架。2025 年，这一切正在改变。我们看到 AI 工具的快速采用与不断演变的语言偏好之间存在相关性。这些变化表明 AI 不仅影响代码编写的速度，还影响开发者使用哪些语言和工具。

2025 年最大的事情之一是什么？Agent 来了。我们数据中的早期信号开始显示它们的影响，但最终指向一件关键的事：我们才刚刚开始，预计未来几个月和几年会有更多活动。

2025 年 GitHub 现状：创纪录增长的一年

核心增长指标

  • 平均每秒 1 位开发者加入 GitHub（同比增长 23%）
  • 每分钟创建约 230 个新仓库（年增 1.21 亿）
  • 公开和开源仓库总计 3.95 亿（年增 7200 万）

2023 年，GitHub 开发者数量突破 1 亿，从 5000 万增长到 1 亿用几乎用了同样的时间。但仅过去一年就改写了这条曲线，实现了有史以来最快的绝对增长。如今，超过 1.8 亿开发者在 GitHub 上构建。

那么，"平均每秒超过一位新开发者加入 GitHub"实际意味着什么？

  • 开发者正在向 GitHub 聚集：超过 3600 万开发者在一年内加入 GitHub（同比增长 23%），确认 GitHub 作为协作主要中心的地位
  • AI 采用即刻开始：我们看到近 80% 的 GitHub 新开发者在第一周内就使用 GitHub Copilot，这表明 AI 现在是新程序员的期望标配
  • 人才增长在地理上多元化：每分钟约 25 位开发者从亚太地区加入，约 12 位来自欧洲，约 6.5 位来自非洲和中东，约 6 位来自拉丁美洲。仅印度今年就新增超过 500 万开发者

GitHub Copilot 加速了增长曲线

历史上，开发者注册和仓库创建遵循可预测的年同比模式。2024 年 12 月 Copilot Free 的推出在全球范围内加速了这些曲线，让数百万人首次获得 AI 驱动的工作流。最终结果？我们典型的增长模型被彻底颠覆。

私有和公开仓库扮演不同但相互依存的角色

2025 年，81.5% 的贡献发生在私有仓库中，而 63% 的仓库是公开的。这一分布凸显了 GitHub 的双重角色：大多数日常工作在私有项目中进行，但依赖于公开开源中的库、模型和框架。

2025 年度视角 贡献数 占比 说明
私有仓库 49.7 亿 ≈81.5% 企业和团队级协作正在 GitHub 上进行
公开仓库 11.2 亿 ≈18.5% 工作量较小，但这些项目提供了驱动更广泛生态系统的库、模型和工作流

关键数字

  • 180M+ 开发者现在在 GitHub 上
  • 630M 总仓库，2025 年新增 1.21 亿，是我们有史以来最大的一年
  • 新增 5800 万私有仓库（增长 33%），凸显了公众视野之外的活动
  • 开源和公开项目占 GitHub 所有仓库的多数。63% 的仓库是开源或公开的

开发者生产力：交付更多，等待更少

开发者生产力指标

  • 2025 年 7 月，550 万个 issue 在公开和私有项目中被关闭
  • 2025 年每月合并 4320 万个 Pull Request
  • 截至 2025 年 3 月，Jupyter Notebook 同比增长 75%

2025 年是 GitHub 历史上最活跃的 12 个月期间，公开和开源项目的贡献超过 11.2 亿次。根据 SPACE 框架（一个关注开发者满意度、绩效、活动、沟通和效率的模型），这一增长反映了创纪录的开发者活动水平。

势头在 2025 年初加速，与 3 月 Copilot coding agent 预览版和 4 月 Copilot code review 推出时间吻合。代码推送是这波增长的主要驱动力，2025 年超过 9.86 亿次 commit（同比增长 25%）。

  • Pull Request 创建：+20.4%（4750 万 vs 3950 万）
  • Issue 创建：+11.3%（1750 万 vs 1570 万）
  • Issue/PR 评论：基本持平（+0.35%）

AI Agent 进入主流

  • AI 代码审查帮助开发者：72.6% 使用 Copilot code review 的开发者表示它提高了他们的效率
  • 开发者正在使用 AI 和 Agent 工具更快地构建和分享工作：2025 年数据显示快速原型开发和实验有所增加

什么是 Vibe Coding？

2025 年一个值得注意的趋势是"vibe coding"。由 Andrej Karpathy 首创，vibe coding 成为一种开发者工作流的简称，从一个想法开始，直接跳到可运行的概念验证（通常在一个晚上完成，由 AI 自动补全和可复制粘贴的云工具驱动）。

如果 AI 辅助工具继续降低入门门槛，我们可能会看到编程素养大幅扩展。

展望 2030 年全球开发者格局

分析结果表明，印度将继续扩大领先优势，到 2030 年达到 5750 万开发者，占全球预计注册量的三分之一以上。美国将成为第二大社区，预计超过 4000 万开发者，而巴西（1960 万）、日本（1170 万）和英国（1100 万）跻身前五。

关键要点

  • 今年加入 GitHub 的新开发者中，三分之一来自 2020 年不在全球前十的国家

  • AI 热潮是全球性的：GitHub 上生成式 AI 项目的贡献者数量持续快速增长，这些贡献者遍布全球

安全：从"左移"到默认安全

核心安全指标

  • 84.6 万个仓库现在配置了 Dependabot（同比增长 137%）
  • 15.1 万个仓库被标记为访问控制漏洞（同比增长 172%，现为 CodeQL 头号警报）
  • 严重漏洞修复速度加快 30%

过去一年，严重漏洞的平均修复时间提高了 30%，因为补救开始跟上更快的软件开发速度。

自动化正在推动这一加速。Dependabot 使用量翻了一番多（84.6 万项目，同比增长 137%），而 Copilot Autofix 等 AI 工具每月在数千个仓库中解决常见的 OWASP Top 10 问题。2025 年，收到严重警报的仓库减少了 26%。

新风险出现：访问控制漏洞超过注入成为最常见的 CodeQL 警报，在 15.1 万多个仓库中被标记（同比增长 172%）。这很大程度上源于 CI/CD 管道中配置错误的权限和 AI 生成的脚手架跳过关键身份验证检查。

更快的修复时间

2025 年，严重漏洞修复速度加快 30%，收到严重警报的仓库减少 26%。平均修复时间从总计 37 天缩短到 26 天。

配置和编码化安全

2025 年 CodeQL：访问控制漏洞激增

访问控制漏洞超过注入成为头号 CodeQL 警报。这指向一个更广泛的问题：身份验证和授权对开发者和 LLM 来说仍然困难。

到 2025 年中，开发者每月在 6000 多个仓库中接受 AI 生成的访问控制漏洞修复。Autofix 在注入（3100 个项目）、不安全设计（2300 个项目）和日志/监控失败（3500 个项目）方面也获得了关注。

为什么 TypeScript 在 2025 年获胜

  • 按数字：TypeScript 在 2025 年 8 月以 2,636,006 月度贡献者（同比增长 105 万；+66.6%）完成第一
  • 类型契约扩展 AI 辅助团队：类型系统减少歧义，在生产前捕获 LLM 错误
  • 框架默认使用 TypeScript：Next.js 15、Astro 3、SvelteKit 2、Qwik、SolidStart、Angular 18 和 Remix 默认生成 TypeScript 代码库
  • 类型系统帮助更早识别 LLM 生成的编译错误：2025 年的一项学术研究发现 94% 的 LLM 生成编译错误是类型检查失败
  • 低入门门槛：Vite、ts-node、Bun 和 IDE 自动配置等工具隐藏样板代码

生成式 AI 和 Agent 工作流成为普通工程

核心 AI 指标

  • 430 万 AI 相关仓库（自 2023 年以来几乎翻倍）
  • 110 万公开仓库导入 LLM SDK（同比增长 178%）
  • 190 万 AI 项目月均贡献（同比增长 76%）

去年我们看到 AI 从实验转向主流。2025 年，它成为日常工作流的一部分。约 80% 的 GitHub 新用户在第一周就尝试 Copilot，表明 AI 不再是需要成长才能使用的高级工具，而是默认开发者体验的一部分。

关键要点

  • 生成式 AI 正在成为基础设施：超过 110 万公开仓库现在导入 LLM SDK（同比增长 178%），由 105 万多贡献者和 175 万月度 commit 支持（自 2023 年以来增长 4.8 倍）
  • 增长在日历年初激增，然后随着项目从实验转向交付而正常化
  • 开源中的 AI：半数（50%）开源项目至少有一位维护者使用 GitHub Copilot
  • 从原型到生产转变的早期证据：Python 代码在 2025 年中加速，而 Notebook 增长趋平

这告诉我们什么

  • 软件基础设施在速度上超过其他一切：全新的生成式 AI 仓库（≤1 年）正在积累其他项目花十年才能积累的 star 数
  • 标准正在实时出现：Model Context Protocol（MCP）的快速崛起显示社区正在围绕互操作性标准凝聚
  • AI 正在重塑经典工具：像 ollama 和 ragflow 这样的项目展示了本地推理和 AI 增强管道如何从概念验证进入主流开发者工作流

AI 也在帮助开发者修复代码

GitHub Copilot Autofix 在 2025 年做出了可衡量的改进：

  • 访问控制漏洞增长最快，到 2025 年中，每月有 6000 多个仓库接受修复
  • 安全日志和监控失败、注入、不安全设计和配置错误修复也急剧攀升
  • Autofix 正在解决最常见的 OWASP Top 10 问题

如何保持领先

  • 预期 AI 库成为"管道"：如果你的技术栈不能加载模型或将上下文传入其中，你很快就会感到遗留束缚
  • 超越 Notebook：尽早打包你的实验以与他人分享
  • 关注工具链，而不仅仅是模型：下一个生产力飞跃可能来自 LLM 原生编辑器、shell 和测试运行器
  • 为互操作性构建：像 MCP 和 Llama 衍生协议这样的标准正在跨生态系统获得势头

结语

三年前，我们说 AI 不会取代开发者——它会为生态系统带来更多人。数据现在证明了这一点：GitHub 上的活动达到创纪录水平，贡献、仓库和实验比以往任何时候都多。

过去一年标志着历史性里程碑：

  • 印度超越美国成为 GitHub 上公开和开源项目的最大贡献者基础。它也有望在未来几年超越美国开发者人口，凸显开发者社区已经变得多么全球化
  • TypeScript 首次成为使用最多的语言，超越 Python 和 JavaScript，预示着现代软件构建方式的代际转变
  • 开源仍然是基础。公开项目提供驱动大多数私有开发的库、模型和工作流。这个生态系统的强度和维护它的维护者将决定下一波软件创新能走多远、多快

2025 年的故事不是 AI 对抗开发者。它是关于 AI 时代开发者的演变，他们编排 Agent、塑造语言、驱动生态系统。无论他们选择哪个 Agent、IDE 或框架，GitHub 都是一切汇聚的地方。
