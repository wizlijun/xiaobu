媒介与功能：产品战略中的选择

在思考产品设计时，我们常常面临一个问题：什么时候应该"做功能"，什么时候应该"做媒介"？这个问题背后涉及对产品本质的理解。本文整理了 McLuhan 媒介理论与现代技术产品设计的核心观点，结合工作流与结构思维，探讨媒介与功能在产品战略中的区别。

什么是真正的"媒介"

媒介不是"功能"也不是单纯的内容

Marshall McLuhan 最著名的论断是：

"媒介即讯息" — The medium is the message

他强调的是：媒介本身的存在、结构与形式，会比它承载的内容对人类感知、思维方式和社会行为产生更深远的影响。即使内容不变，媒介不同，人们的理解、行为和组织方式也会不同。

例如：

  • 书本这种媒介塑造了线性思维、个体阅读习惯
  • 电视这种媒介塑造了视觉节奏与"节目化"文化
  • 网络这种媒介改变了连接、反馈和参与机制

媒介定义（McLuhan 观点）

媒介 = 任何扩展人类能力、改变沟通流程与经验的技术或形式。它不只传递信息，更重塑人类的工作方式、社会关联和感知结构。

这也是为什么 McLuhan 会把光 bulb、语言、书写甚至火车等都视为媒介，而非只限于媒体产业的概念。

媒介 vs 应用程序 / 功能 — 在技术产品里的区别

在软件领域，当我们说一个系统是 应用程序（Application） 时，它通常意味着：

  • 有明确任务或功能
  • 面向解决某些具体问题
  • 功能边界相对固定
  • 用户消费它的功能

而当我们说一个系统是 媒介（或平台）：

  • 它是一个开放的结构、生态和表达空间
  • 不是"给你做某件事情"，而是"让你在其上创造你想做的事情"
  • 用户不仅消费内容，还能生成、改造和组合
  • 创作者、参与者和消费者之间角色模糊甚至互换

换句话说，它不是单一功能的堆叠，而是一个创造性的基础设施 / 架构。

类似的产品属性对比：

平台（Platform） = 基础 + 扩展 + 自定义能力

应用（Product / Application） = 目标导向的功能集合

（平台允许第三方创造 / 扩展；应用则是封闭、定义明确的功能集）

媒介 = 一个支持用户持续创造与参与的底层结构（可塑空间），而不是固定任务的功能模块。

LLM / AI 作为新的媒介？

从这个角度看：

  • "AI 变成新的 Medium，不同的人在上面创作、交易"
  • "不是仅仅写死的功能，而是给出基础工具 / 结构供人自定义"

这是符合媒介理论核心精神的。

AI/LLM 本质上提供了一个表达与生成的基础结构

这种结构具有：

  • 开放性：用户可以提出无限种不同的 prompt
  • 可组合性：不只是使用它的既定功能，而可以与工具 / 流程 / 数据结合起来创造新系统
  • 可嵌入工作流：从教育、创作到自动化、协作构建新的工作结构

换句话说，AI 在这里不仅是"做功能"，而是：

一个能够塑造用户思维惯性、工具组合方式、信息流结构的人机共同体媒介。

这种媒介的意义远超单一"应用功能"。它改变了人类如何表达想法、探索未知、组织任务和产生知识。

这是 McLuhan 所强调的：媒介本身改变了人类的感知节奏和认知结构，AI 作为媒介，其效果体现在重塑人类的工作方式与认知工具，而非只是内容生成。

功能 vs 媒介：产品战略怎么抉择？

从工作流 / 结构思维来看：

工作流中的确定性结构（tools / categories / layouts / behavior）创造流畅的创造过程。产品成为结构中的重要部分，才有不可替代的价值。

这其实和"媒介"的概念高度相关。

媒介 vs 功能

特征 功能（单体 Feature） 媒介 / 平台
角色 完成功能任务 支撑多种行为 & 表达方式
控制权 工具方定义功能 用户可塑造、组合和扩展
价值方向 给用户提供解决方案 改变用户行为模式和工作方式
生命周期 随需求改变迭代 持续演化、积累生态
用户依赖 因功能需求而用 因结构 / 生态 / 长期价值黏住

结论

一个产品如果只是"功能堆叠"，它就是应用；但如果这个产品是一个可以承载、塑造、促进用户创造与新行为的空间，它才是媒介。

你的产品在做什么，应该往哪边靠？

根据 McLuhan 学派和现代平台设计实践，可以总结：

做功能是基础

功能让产品具备解决用户即时需求的能力

分析需求 → 定义功能 → 提供可执行路径

做媒介是战略

媒介让产品成为用户思维 / 行为的环境和基础

设计结构 → 提供可组合性 → 支持自定义生态

什么时候做媒介？

  • 用户的核心需求是创造而不是仅消费
  • 用户场景复杂、无限延展性强
  • 你希望用户能在你的系统上构建新的流程、产品、表达
  • 你的价值并非来自特定结果，而是来自用户创造路径本身

这也是为什么很多现代成功产品（例如：操作系统、web 浏览器、低代码平台、AI 模型接口、社区创作平台）都强调：

用户不是"被动使用"，而是介入创造整个生态。

这和媒介理论中的观点一致 —— 媒介是塑造人类的思考与交互结构，而不仅是提供结果。

总结

  1. 媒介 ≠ 功能
    功能是"具体解决方案"，媒介是"赋予用户创造与表达能力的结构与环境"。
  2. McLuhan 的媒介理论告诉我们
    媒介塑造人的感知与行为，而不是简单传递内容。
  3. 如果你希望产品成为媒介，就要
    • 关注用户的创造性空间
    • 设计开放的可组合结构
    • 支持可扩展 / 可定制的生态
  4. AI / LLM 能成为新的媒介
    因为它不仅提供功能，还改变了人类表达、构建信息与组织行为的方式。

产品战略框架

1. 明确产品愿景与目标：成为用户创造与参与的媒介

目标是让产品不仅是用户完成某项任务的工具，而是能够赋能用户创造、分享、参与并扩展其行为的空间。此时，产品本身不仅仅是一个"功能"，而是一个能够自我延展与演化的平台。

关键点：

  • 产品不是预定义的解决方案，而是开放的互动空间，允许用户在其中塑造与创造
  • 核心价值：用户创造能力、可塑性、生态扩展性

2. 产品设计思维：构建可以自由组合的开放架构

  • 模块化设计：提供可以自由组合、定制和拓展的基础结构。这些模块不只局限于简单的功能，而是为用户提供广阔的创造空间
  • 自定义接口：允许用户根据自己的需求和目标调整产品的表现形式。例如，可以通过 API、插件等形式让用户扩展原有的功能
  • 低代码 / 无代码平台：让没有编程经验的用户也能自定义和开发应用，从而增加平台的可用性和适应性
  • 透明化与开放性：保证平台的透明度和开放性，允许用户与开发者之间的互动更具创意性与灵活性

3. 核心技术架构：构建为创作与协作设计的底层平台

  • 人工智能与自动化支持：利用 AI 技术提升平台的智能化支持，包括自然语言处理、个性化推荐和协同创作工具等。AI 不仅是一个功能工具，更是推动用户与平台之间互动的核心媒介
  • 协作与社交功能：强化平台的社交属性，让用户可以在平台上协作、分享与交易内容，打破单一创作者与消费者的壁垒
  • 灵活的数据处理和储存：支持大规模用户生成内容和多样化数据格式，确保平台能够处理不断增长的用户需求

4. 用户体验与参与：激发用户创作的驱动力

  • 用户教育与引导：提供丰富的学习资源和创作指南，帮助用户快速上手并发挥平台的创造潜力
  • 激励机制：通过积分、交易、奖励等机制激励用户创作和参与互动。例如，用户生成的内容可以进行交易或获得平台回报
  • 内容展示与反馈机制：强化社交分享和反馈环节，展示用户创作的作品，鼓励互动与传播

5. 生态建设：创建可持续的创作与交易系统

  • 开放市场：允许用户交易创作内容、插件、模块等，鼓励平台内外的经济活动。用户可以在平台上建立自己的"商店"或创作集市
  • 合作与共建：鼓励用户与开发者、设计师等合作，共同开发平台的功能与扩展模块，从而形成一个共同发展的社区生态

产品路线图建议

阶段 1：核心产品开发与基础设施建设

目标：完成产品的最小可行产品（MVP），并构建稳定的基础架构和核心功能。

  • 模块化架构的初步搭建，包括基础功能和接口
  • 提供基本的创作工具与自定义功能
  • 启动核心 AI 能力（如自然语言生成、智能推荐等）
  • 开设社区交流与反馈平台，让用户能互相学习与创作

阶段 2：用户教育与功能扩展

目标：扩展功能，增加用户参与度，并为创作者提供更多的自由。

  • 通过培训、教程、在线课程等引导用户快速上手，提升其创作能力
  • 引入低代码 / 无代码工具，让非技术用户也能进行自定义
  • 推出社交功能，让用户能分享创作内容，鼓励互动与合作
  • 强化 AI 在用户创作中的辅助作用，提供更加智能化的支持
  • 开启创作市场，允许用户交换内容或工具，创造激励

阶段 3：生态发展与多元化创作

目标：打造一个完善的创作与交易生态，成为用户创作与交换的平台。

  • 拓展创作工具种类（如更多的模块、模板、插件等），允许用户根据需求自由组合
  • 加入可交易的创作资产，支持内容、插件和创作工具的市场交易
  • 建立开放接口（API），让其他平台与应用能够集成到你的生态系统中
  • 增加平台的多语言与国际化支持，吸引全球用户参与

阶段 4：持续优化与生态成熟

目标：优化用户体验，增强平台的可扩展性与用户粘性，形成长期可持续发展生态。

  • 通过用户反馈与数据分析，不断优化平台功能和体验
  • 强化平台的社交与协作功能，鼓励用户与其他用户共同创作
  • 增加更多合作伙伴和第三方开发者，扩大平台的技术和创作能力
  • 继续探索与发展新兴技术（如区块链在内容创作交易中的应用、更加智能化的 AI 支持等）

结语

"结构"和"媒介"的关系提示我们，产品的最终目标不仅是完成某个任务，而是成为人类活动的基础设施，让每个用户都可以在其上进行创新、协作与发展。通过上面的战略框架与路线图，产品可以逐步演变为一个支持创作、交流与交易的开放平台，从而为用户创造更多的长期价值。
