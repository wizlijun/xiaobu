开场愿景：从“页面”到“智能体”
核心观点： 行业正在经历从“页面（Pages）”到“智能体（Agents）”的根本性转变。构建 AI 应用的复杂性，如同 2014 年的 Web 开发，亟需一个统一的解决方案。
"The world is moving from pages to agents... It felt obvious we needed this conference because we've seen this movie before."
  • 面临的痛点: 开发者在 AI 应用开发中，花费大量时间处理模型切换、SDK 兼容、速率限制和计费对账等琐碎问题，而非专注于核心创意。
  • Vercel 的解决方案: 推出 Vercel AI Cloud，一个集成了 AI 优先工具、安全、可观测性和计算能力的一站式平台，旨在简化智能应用的开发与部署。
  • 核心组件 - AI SDK:
    • 作为 AI Cloud 的入口，统一了各大模型提供商的 API。
    • 数据：每周下载量接近 400 万次，已成为构建 AI 应用的行业标准。
    • 它并非“有损抽象”，而是完整保留了底层模型的全部能力。
  • 智能体的定义: 一个能在循环中调用工具（Tools）以追求特定成果的 AI 模型。其生命周期为：接收提示 (Prompt) → 推理 (Reason) → 行动 (Act)。
  • 低代码/无代码的局限性： 适合原型设计，但在生产环境中显得脆弱，缺乏版本控制、CI/CD 和可扩展性。卓越的工程能力才是决胜关键。
AI SDK 6.0 (Beta): 更强大的智能体构建工具

AI SDK 源于 Vercel 团队在构建明星产品 v0 (GenUI 工具) 时遇到的实际挑战。最新发布的 6.0 Beta 版本带来了两大革命性更新。

新特性 1: 智能体抽象 (Agent Abstraction)

定义一次智能体（模型、工具、行为），即可在任何地方（交互式 UI 或后台任务）轻松复用，极大提升了代码的模块化和重用性。

新特性 2: 工具执行审批 (Tool Execution Approval)

原生支持“人机协同 (Human-in-the-loop)”模式。只需一行配置 needsApproval: true，智能体在执行关键工具（如发帖、删库）前会暂停并请求人类批准，无需复杂的变通方案。

  • 现场演示: 快速构建了一个“社交媒体内容生成智能体”，该智能体能读取 GitHub PR、进行网络搜索、模仿写作风格，并在发布前请求人类批准，完整展示了新特性的强大之处。
  • 可用性：AI SDK 6.0 Beta 版已发布，可立即使用。稳定版预计于今年晚些时候推出。
Vercel AI Cloud: 为生产环境打造的 AI 基础设施

Vercel AI Cloud 提供了一系列专为构建和运行大规模、高可靠性智能体而设计的底层基础设施（Primitives）。

  • Vercel Agent 新技能 - AI 调查:
    • 除了已有的 AI 代码审查，Vercel Agent 新增“AI 调查”能力。
    • 当监控到异常事件（如半夜流量激增），它会自动进行根本原因分析，并判断是否需要唤醒人类工程师，旨在提升开发者的“睡眠质量”。
  • 底层设施 1: AI Gateway (AI 网关)
    • 使用单个 Vercel API 密钥即可访问超过 100 种模型（包括 OpenAI, Anthropic, Groq 等）。
    • 提供自动故障转移（如 Claude 可在 AWS Bedrock, Vertex AI 和 Anthropic 自身之间切换），保证服务稳定性。
    • 对模型 Token 不加价（Zero upcharge），只收取和官方一致的费用。
  • 底层设施 2: Fluid Compute (流式计算)
    • 一种为 AI 时代设计的 Serverless 计算模式。
    • 核心创新：只对 CPU 实际使用时间计费，外加极低的内存占用费。在等待模型响应、工具调用或人类输入时的空闲时间不计费，大幅降低 AI 应用成本。
  • 底层设施 3: Vercel Sandbox (沙盒)
    • 在隔离的 MicroVMs 中安全执行不可信代码（如 LLM 生成的代码）。
    • 运行于 Fluid Compute 之上，兼具安全与成本效益。
  • Python 全面支持: Vercel 现已原生支持 FastAPI 和 Flask 框架的零配置部署，并推出了官方 Python SDK。
WDK: 开源的工作流开发套件
核心问题： 智能体通常是长时运行、多步骤的，传统 Serverless 架构难以处理这种持久化、高可靠性的需求。开发者需要手动编排队列、数据库等复杂组件。
全新开源框架: Workflow Development Kit (WDK)

WDK 将复杂的分布式系统编排能力，压缩为简单的代码指令，让开发者能像编写单体函数一样编写高可靠的工作流。

  • 核心指令:
    • use workflow: 将一个普通函数标记为持久化的工作流。
    • use step: 将函数内的某个操作（如 API 调用）标记为一个可靠的步骤。
  • 工作原理:
    1. WDK 自动将每个 step 编译为独立的、可通过队列调用的 API 端点。
    2. 工作流执行到 step 时会暂停，在后台执行该步骤。
    3. 步骤失败会自动重试。成功后，工作流从事件日志中恢复状态，继续执行，保证了最终一致性。
  • 内置优势:
    • 高可靠性： 自动重试和状态持久化。
    • 原生可观测性： 自动记录每次运行、每个步骤和 LLM 调用，无需额外配置。
    • 无缝版本控制： 在 Vercel 上部署是原子性的，不会破坏正在运行中的长时工作流。
  • 可用性：已通过 npm install workflow 发布，支持 Next.js, Nitro, Hono 等框架。
落地应用、生态系统与未来展望

Vercel 不仅提供了工具，还通过实际案例和生态建设计划，推动智能体在各行各业的落地。

  • 新发布: Agent Marketplace (智能体市场)

    一个发现、安装和连接预构建智能体和工具的市场，合作伙伴包括 CodeRabbit, CodiumAI, Sourcery 等。

  • 如何找到构建智能体的最佳切入点？ 去采访你的员工，问两个问题：
    1. 你工作中最讨厌的部分是什么？
    2. 哪些重复性的“忙碌工作”你希望再也不用做？
  • Vercel 内部案例与开源项目:
    • 商机处理智能体 (已开源): 自动处理销售线索，过滤垃圾信息、丰富客户资料、起草跟进邮件并同步到 Salesforce，将处理时间从 15 分钟缩短到秒级。
    • 商业智能 Slack 智能体 (已开源): 在 Slack 中用自然语言提问，即可自动生成 SQL 查询并返回数据分析结果，因为它被专门训练以理解公司的业务语义。
    • 滥用行为处理智能体 (未开源): 协助安全团队处理钓鱼、诈骗等滥用报告，使每起案件的处理时间减少了 59%。
  • 新计划: "An Agent on Every Desk" (每个桌面都有一个智能体)

    一项针对企业的深度合作计划。Vercel 将直接派遣工程师，与客户团队合作，帮助他们识别需求并构建第一个成功的、能产生巨大商业价值的智能体。
