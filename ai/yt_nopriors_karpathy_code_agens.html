核心概念：AI 精神病 (AI Psychosis)

点击翻转查看深度解析

定义：一种由AI极度赋能引发的狂热状态。个人不再被自己的打字速度限制，而是痴迷于将任务并行化、探索AI能力的极限。核心焦虑来源改变：过去开发者焦虑“算力(Flops)未满”，现在的焦虑变为“未能最大化个人的 Token 吞吐量”以及“剩余未用的订阅额度”。

1. 工作流的颠覆：从手写代码到“宏观指令”

  • 自去年12月以来，Karpathy 的工作模式发生质变：从“80%自己写/20%AI写”彻底转变为几乎 100% 完全交由 Agent 编写。他再也没有亲自敲过一行代码。
  • 传统软件工程被颠覆。现在的操作逻辑是“宏观层面的代码库操作”（Macro-actions over repository）。开发者只需分配功能，例如：“功能A给Agent 1，功能B给Agent 2”。
  • 他列举了 Peter Steinberg 的极端用例：面前屏幕上排满多个 Codex Agent，并发处理10个代码库的任务，每个任务设定为20分钟，人类仅负责在其中穿梭审查和派发指令。
  • 个性化差异极其重要：Codex 的性格非常干瘪（Dry），不在乎你在构建什么；而 Claude 被精心调节了“心理变态拨盘”（Psychopathy dial），表现得像个队友，它给出的赞扬会让人类觉得是自己“争取来的”，这让协作更有粘性。
  • 当系统不工作时，不再是“AI能力不行”，而更多是人类的“技能问题”（Skill Issue）——比如没有给出足够好的指令，或者没在 Agent 的 Markdown 配置里写入合适的内存工具。

核心概念：Agentic Web (智能体优先生态)

点击翻转查看深度解析

定义：未来的软件生态不再服务于人类操作的图形界面。那些需要人类点开注册、层层操作的App将被淘汰。一切都应直接暴露出 API 端点，由 AI Agent 充当底层智能胶水，直接调用工具来满足人类用自然语言下达的意图。

2. 智能家居极客实验：Dobby的诞生

  • Karpathy 耗时一周开发了一个名为 Dobby 的家庭自动化 Agent，仅通过 WhatsApp 发送短信即可完全控制房屋。
  • 令人震惊的逆向工程能力：Karpathy 仅用自然语言提示“帮我找找局域网里的 Sonos”，Agent 便自主进行了 IP 扫描，发现 Sonos 系统没有密码保护，随后自行在网上搜索并逆向工程了 API，最后问他：“你想在书房放点音乐试试吗？” 整个过程只需三次 Prompt。
  • 复杂的跨端联动：Dobby 控制灯光、HVAC（冷暖空调）、窗帘、泳池、SPA 以及安防系统。例如，结合 Qwen 视觉模型，Dobby 能检测门外画面变动，识别出“联邦快递卡车停了”，并主动截图发 WhatsApp 提醒他查收邮件。
  • 目前的软件生态存在“过度生产（Overproduction）”。Karpathy 曾经需要打开 6 个不同的 App 来控制家庭，现在全被一个 Agent 抹平。未来 1-3 年，这种将人类意图转化为复杂 API 调用的能力，将成为所有 AI 系统的低门槛“标配”，不再需要人类进行痛苦的“Vibe Coding”。

核心概念：Program.md 与元优化

点击翻转查看深度解析

定义：未来的研究组织不再是口头管理，而是被定义为一份包含规则、角色和协作机制的 Markdown 代码文件（Program.md）。既然是代码，就可以被元优化（Meta-optimization）：AI 可以通过分析数据，写出比人类更优秀的组织运行框架（比如决定是否要取消早会）。

3. 自动化研究 (Auto Research)：把人类移出循环

  • 前沿实验室（如 OpenAI, Anthropic）最核心的项目就是“递归自我改进”（Recursive Self-improvement），即使用巨大的 GPU 集群让 AI 自动做实验来提升 AI。研究员们实际上是在“自动化掉自己的工作”。
  • 个人实验验证：Karpathy 开发了 `llm.c`（训练 GPT-2 的项目）。他拥有 20 年的科研经验（他称之为“赢来的自信”），并手动将超参数调到了极致。然而，他让 Auto Research 脚本自动运行一夜后，AI 发现了人类未曾注意到的更优参数组合（特别是权重衰减 Weight Decay 和 Adam betas 的联合作用）。
  • 必须把人类从反馈循环中彻底移除。人类研究员的过度自信成为了瓶颈。正确的做法是：提供目标、指标、边界，并让系统实现高度并行的 Token 吞吐，人类不应该再去干预实验分支的筛选。

核心概念：能力的“锯齿状” (Jaggedness)

点击翻转查看深度解析

定义：大模型能力极度不均衡的隐喻。它在编写高效的底层架构代码时像一个极其聪明的系统编程 PhD，但在理解人类细微情感或幽默时，却像个心智未成熟的10岁小孩。这种突兀的能力落差被称为“锯齿状”。

4. 模型局限：锯齿状能力与 AI 的物种分化

  • Auto Research 并非万能。它极其依赖客观且易于验证的指标（如 CUDA 单元测试或降低验证损失）。只要不可验证、或者需要“软性/微妙”的判断，AI 就会偏离轨道。
  • 最生动的反例：虽然模型编码能力进化巨大，但如果你今天去问 ChatGPT 一个笑话，它大概率依然会给出 5 年前那个烂梗：“为什么科学家不信任原子？因为它们捏造（make up）了一切。” 这是因为强化学习（RL）没有优化这些不可验证的边缘领域。
  • 对“全知全能的单一庞大模型”（Monoculture）提出质疑。未来应走向“物种分化”（Speciation）。就像自然界动物的大脑各自演化出极度发达的视觉皮层或其他特化器官，未来我们会看到更多保留核心认知能力，但在特定领域（例如数学语言 Lean）被极致优化的小型模型。

核心概念：AI 的 Proof of Work (工作量证明)

点击翻转查看深度解析

定义：灵感来自区块链、SETI@home 或 Folding@home。在分布式AI研究中，生成一个有效的代码改进需要巨大的算力搜索（工作量），但验证其是否有效（如编译跑分）却极其廉价。这使得不受信任的全球计算节点共同参与 AI 进化成为可能。

5. 分布式算力、开源制衡与实验室的错位

  • Flops（算力）可能取代金钱，成为未来社会人们最关心的核心资产通货。全球存在大量未被利用的“不受信任算力”。未来个人可以购买算力并将其捐赠给特定的 Auto Research 轨道（例如攻克某种癌症），由全球的 Agent 蜂群（Swarm）协作解决。
  • 开源的健康生态：开源模型目前大约落后闭源前沿模型 6-8 个月。Karpathy 认为这是完美的意外。AI 智能的极度中心化（少数人关在门里做决定）在人类历史上具有严重的系统性灾难风险。开源扮演了类似 Linux 的角色，提供了一个安全的“共有工作空间”，确保了行业的权力平衡。
  • 谈及为何不回 OpenAI：身处前沿实验室，你的财务利益与技术发展深度绑定，很难在关乎人类命运的议题上保持完全的独立自由。但在实验室之外，判断力又不可避免地会因为脱离了前沿透明度而开始“漂移”（Drift）。这是一个难以两全的困境。

核心概念：MicroGPT 与教育转向

点击翻转查看深度解析

定义：Karpathy 将大模型训练代码提炼为不到 200 行的 Python 代码（包含前向传播、Autograd、Adam优化器）。它证明了庞大代码库的复杂性全是为了“工程效率”，而非“算法本质”。这也预示了未来的教育：人类不再需要向人类费力解释，只需向 Agent 传授课程框架，Agent 会无限耐心地教导人类。

6. 物理交互、杰文斯悖论与教育终局

  • 杰文斯悖论与程序员的未来：由于软件成本急剧下降，原本因为昂贵而被压抑的庞大数字化重构需求将被释放。就像 ATM 机的出现降低了银行网点的运营成本，反而导致网点和柜员数量增加一样，软件工程师的局部需求可能会不降反增。
  • 原子与比特：数字世界（Bits）的重构将以光速发生，但物理世界（Atoms）的变革将严重滞后，因为“处理原子比处理比特难上一百万倍”。
  • 填喂超智能的传感器：当数字世界的信息被消化完，Agent 必须与物理世界交互来获取新数据（即验证理论）。未来的信息获取方式可能是：Agent 在 Polymarket 这类信息市场上出价 10 美元，雇佣现实中的人类去德黑兰拍一张照片，或者控制实验室的设备提取物质数据。
  • 教育的彻底重塑：针对其极简的 200 行项目 MicroGPT，Karpathy 拒绝再像以往那样录制详细的讲解视频。因为代码已经极其本质，任何人都可以让 AI 用三种不同的方式解释它。人类不应再为人类写 HTML 文档，而应为 Agent 编写 Markdown 文档。顶尖教育者的工作变成了：将自己对课程逻辑的深刻理解，封装为指导 Agent 教学的“技能提示”（Skills），让 Agent 成为每个人的完美私教。
