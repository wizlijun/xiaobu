访谈深度摘要

1. 从倦怠到觉醒：重写规则

事实 Peter在运营PSPDFKit 13年后感到精疲力尽。最初尝试GPT-Engineer类工具时并不感冒，直到他尝试用 Gemini Studio 2.5 和 Opus 3.5 重写一个半途而废的项目。

观点 真正的“Aha Moment”发生在他意识到：以前软件开发受限于时间，现在受限于想象力。 他将整个项目拖入AI，一小时内完成了过去需要数周的工作，这种“多巴胺冲击”让他意识到世界变了。

2. Open Claw 的诞生与疯狂的社区

事实 项目起初只是Peter的个人游乐场，旨在通过AI自动处理WhatsApp消息。他在Marrakesh旅行时，用它自动翻译菜单、回消息。

事实 “不可能”的语音消息事件： Peter发送了一条没有后缀的Opus音频文件给Agent。Agent自己检查文件头，发现是Opus格式，自己编写脚本调用本地的ffmpeg转换，发现没有Whisper模型，于是搜索环境找到OpenAI Key，用curl调用API完成了转录。这一切都在没有任何预编程的情况下自主完成。

ClawCon 在旧金山举行，吸引了上千人，证明了这不仅仅是代码库，而是成为了一种文化现象。

3. 开发范式的彻底转移

观点 Peter 提出 "Shipping code without reading it"（不读代码直接发布）。

大部分代码只是数据的搬运（Data transformation），是枯燥的。只要Agent理解了高层意图，且系统架构是为了Agent优化（Self-modifying software），人类不需要纠结于实现细节。他现在更多是在与AI“对话”，而非结对编程。

4. 安全与开源的冲突

事实 Peter曾在Discord上公开运行没有任何沙盒保护的Bot，结果并没有被注入恶意代码，反而证明了模型的鲁棒性。

观点 安全界对Open Claw批评不断（如暴露本地端口），但Peter认为这是为了让黑客能自由探索（Hacker's Paradise）。他最终引入了沙盒机制（Mac Studio上的"Castle"），但强调不能因噎废食，要允许创造性的误用。
