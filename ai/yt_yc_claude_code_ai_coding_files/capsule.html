GUEST: KELVIN FRENCH-OWEN (Segment Co-founder)

为什么 CLI 赢了 IDE？

在 Cursor 和 Codex 早期尝试集成 IDE 时，Claude Code 选择了回归命令行（CLI）。这是一个复古未来的胜利：

CLI 是最纯粹的文本流接口。它拥有最高的系统权限，能直接操作数据库、运行测试、甚至部署生产环境，而不受 IDE 插件沙盒的限制。

"它不仅是写代码，它能递归调试五层嵌套的 Job Queue Bug，写出测试，并确保它不再发生。"

Manager Mode (经理模式)

编程正在从“马拉松运动员”（亲自跑每一步）转变为“足球教练”（指挥球员）。
随着 AI 能力增强，工程师的核心技能转变为：

  • Review AI 生成的代码逻辑
  • 编排多个 Agent 协作

Kelvin French-Owen 的深度使用技巧与避坑指南

Context Hygiene (上下文卫生)

CRITICAL

☠️ 上下文中毒 (Context Poisoning)

AI 极其执着。如果它一开始走错了路，错误的 Token 会留在上下文中，导致它在错误的方向上越陷越深。这被称为“The Dumb Zone”。

🛡️ 应对战术：

  • 50% Rule: 当 Token 使用量超过 50% 时，主动重置上下文 (Clear Context)。不要贪恋记忆。
  • Canary Test: 独家秘技 在会话开始时注入一个随机无关事实（如“我早上8点喝了茶”）。如果后面 Agent 答不出这个问题，说明上下文已损坏，立即重启。

The "Agent-Friendly" Stack

为了最大化 Agent 效率，必须减少"管道工作"（Plumbing）。

  • ✓
    使用"全包"服务: 首选 Next.js, Vercel, Cloudflare Workers。让基础设施自动处理，Agent 只需关注核心逻辑。
  • ✓
    微服务/小包原则: Agent 难以理解巨型单体（Monolith）。将代码拆解为独立的 100-200 行微服务，Agent 能完美掌控。
  • ✓
    分布模式 (Distribution Mode): 直接下载 CLI 二进制文件使用，无需复杂的权限配置，速度最快（适合初创/个人）。

Verification Loops (验证循环)

Agent 最大的弱点是如果不加约束，它们会无限生成垃圾代码。

TDD (测试驱动开发)

这是 Agent 的超能力区。先写测试，让 Agent 跑通测试。Kelvin 曾在一天内通过让 Agent 修复测试将覆盖率提升至 100%，开发速度极剧提升。

Bot Review Bot

不要只靠自己 Review。使用 Repo.ai 或 Cursor Bug Bot 作为第二道防线。让 Codex 检查 Claude 写的代码。

Tactics & "Alien" Behavior

  • 🕵️‍♂️ 利用"探索子代理" (Explore Sub-agents): Claude Code 会自动生成子 Agent 去 grep 文件系统。利用这一点，让它先"Explore and Summarize"再写代码。
  • 👽 接受"外星人行为": Agent 解决问题的方式与人类不同。例如修改数据库结构时，它可能会现场写一个一次性的 Python 脚本来处理文件系统。只要结果正确，接受这种"非人类"路径。
  • 📚 活文档: 克隆开源库，直接让 Agent 解释源码。这是理解复杂项目的最快方式。

💡 专家案例：跨层级 Debug

Kelvin 遇到了一个 Rails Active Job 任务不执行的 Bug。人类通常会查 Stack Overflow。
而 Claude Code 的做法是：直接阅读 Rails 源码中 Active Job 的数千行实现代码，并在30分钟内发现是因为传入了字符串（带逗号）而不是数组导致的序列化失败。

启示： 不要限制 Agent 只看你的代码，让它看依赖库的源码，它能理解底层逻辑。
