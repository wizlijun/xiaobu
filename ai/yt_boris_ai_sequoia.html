深度挖掘 行业视野

第一阶段：工具论 - Claude Code 的偶然诞生与演进

背景调查：在现场观众中，大多数人使用 CLI（命令行）版本，其次是 Desktop 桌面版，只有少数使用 VS Code / IDE 插件。

  • 并非刻意为之： Claude Code 是由 Anthropic Labs (现由 Instagram 创始人 Mike Krieger 领导第二期) 在 2024 年底作为一个内部孵化项目“偶然”启动的。
  • 跨越“代码补全(Type Ahead)”时代： 当时行业标配是按 Tab 键补全单行代码，但团队预判模型能力即将跃升，决定直接跳过补全，构建“让 Agent 编写所有代码”的全新形态。
  • 熬过“PMF(产品市场契合度) 前”的黑暗： 头6个月由于模型智力不足，产品几乎不可用。为了规避风险，团队故意使用模型最擅长、分布最广的 TypeScript 和 React 构建了 Claude Code 自身的代码库（这部分代码也曾遭到泄露，证实了其架构的简洁性）。
  • 指数级拐点到来： 随着 5 月份 Opus 4 的发布，产品迎来了爆发，随后在 4.5、4.6、4.7 模型中不断飞跃。如今模型已经可以接手任何没见过的诡异语言或框架。
🚀

Product Overhang
(产品悬垂效应)

核心驱动逻辑

指基础 AI 模型已经具备了某种强大的底层能力，但市场上尚未出现能够捕捉和利用这种能力的产品形态。团队正是为了捕捉这种悬垂能力，提前半年构建了产品外壳。

第二阶段：个体论 - 编程已被“解决”与极致工作流

  • 100% AI 接管： 自去年 (2025年) 10月/11月起，模型已100%包揽了 Boris 的代码编写。他不再做技术决策，单日甚至能推进 150 个 PR。
  • 手机统治生产力： 颠覆常识的是，Boris 目前主要通过手机上的 Claude App 侧边栏 (Code Tab)工作。同时运行 5-10 个 Session，随时调动数百个 Agent。夜间会有数千个 Agent 进行深度工作。
🔁

/loop (循环指令)

最简即最强：自动巡检

让 Claude 使用 cron 定时执行任务。Boris 用它跑了几十个循环：自动照看 PR、修复 CI/CD 构建失败、自动 Rebase、排查 Flaky Tests（不稳定测试），甚至每 30 分钟抓取聚类推特反馈。

☁️

Routines (常规任务)

服务器级驻留代理

相当于服务端的 /loop。即便合上笔记本电脑离开，AI 代理依然在云端不眠不休地工作，彻底解绑了开发者的物理设备限制。

⚠️ 前方高能：宏观范式转移

当写代码的成本下降 10 倍乃至 100 倍，被颠覆的不仅是程序员，更是整个 SaaS 商业模式、组织形态和人才定义。

人才需求剧变 组织形态重构

第三阶段：组织论 - 全员皆“兵”的通用人才时代

“这就像 1400 年代欧洲活字印刷术的发明。在它出现之前的几千年，只有被王室雇佣的 10% 的人识字；印刷术出现后 50 年，出版的书籍比过去一千年都多，阅读和书写被彻底平民化。很快，构建软件将像发短信一样普及。”

🔴 旧范式：技术型“全栈”

过去的 Generalist (通才) 仍然局限于工程领域，指的是既懂前端、后端，又懂 iOS 和 Server 开发的技术人员。编码是一项“特权技能”。

🟢 新范式：跨学科领域专家

未来的 Generalist 将是跨学科的。最佳的会计软件编写者不再是工程师，而是顶尖的会计师本人，因为“领域知识”才是最核心的，编码反而是最简单的部分。

Anthropic 的内部组织现状：

  • 在 Claude Code 团队内部，工程经理、产品经理、设计师、数据科学家、财务人员、用户研究员，每一个人都在写代码！
  • 内部已经没有手动编写的代码，甚至所有的 SQL 均由模型生成。
  • AI 之间的“黑话”： 当工程师在写代码时，他们的 Claude Agent 正在后台的 Slack 上自动与其他同事的 Claude 互相交流，以解决未知的信息依赖。真正的壁垒不再是技术，而是这种极致重构的组织流程。
SaaS 重构 商业护城河变迁

第四阶段：商业论 - SaaS 启示录与“七种力量”

面对关于“SaaS 启示录 (SaaS Apocalypse)”的提问，Boris 引用了商业播客《Acquired》推荐的 Hamilton 《七种力量 (Seven Powers)》框架，深入剖析了 AI 如何重构商业护城河：

📉

正在瓦解的壁垒
(Weakened Powers)

旧 SaaS 护城河失效

  • 转换成本 (Switching Costs)： 用户可利用大模型低成本地将工作流从一个平台迁移到另一个平台。
  • 流程优化 (Process Power)： Opus 4.7 已具备“爬山算法(Hill-Climbing)”能力，只要给出目标就能自动迭代流程，企业独有的流程优势被抹平。
🛡️

依然坚挺的壁垒
(Enduring Powers)

AI无法轻易颠覆的护城河

  • 网络效应 (Network Effects)
  • 规模经济 (Scale Economies)
  • 垄断性资源 (Cornered Resources)

这些依赖物理世界积累或多边生态的力量，在 AI 时代变得尤为关键。

🦄

初创公司的黄金时代

10倍速的颠覆

未来十年，能颠覆一切的初创公司数量将增加 10 倍。因为“AI 原生”初创公司没有历史包袱，而传统大公司必须痛苦地对抗内部阻力、重新培训员工并重组旧业务流程。

前沿技术预判

第五阶段：技术终局预判

  • 模型的自治进化： 当前我们还需要学习如何写 Prompt 来让模型并行处理任务（比如 /batch 或子团队）。但以 Opus 4.7 为例，模型已经开始自发决策。比如你让它查数据，它会主动提议：“数据在变动，我开启一个循环，每30分钟在 Slack 给你发个报告吧。”这不再是用户的操作问题，而是模型自身变强。
  • “本地化 vs 云端计算”之争是伪命题： 未来无需开发者去纠结环境在哪里。两年内，模型将自动决定在哪里写代码、建立什么环境。如果它觉得调用本地小模型更优，它就会自行调度，人类不再需要做此类底层决策。
  • 打通知识工作的“任督二脉”：
    • 通过 MCP (Model Context Protocol) 协议，Claude 可以无缝对接 Salesforce、Google Docs、Calendar 等云端知识库。不管是 CLI 还是内测中的 Co-work 都能通用。
    • 对于没有开放 API 的老旧系统，Computer Use (计算机控制) 将作为通用兜底方案。虽然目前动作较慢，但已经非常精准。
  • 未来产品形态的剧透： 随着模型安全审查、指令验证等外壳功能逐渐过时，Anthropic 正在研发如 Claude Design 的新产品，并进一步将大规模并行 Agent (Loop/Batch) 作为一等公民深度整合。
