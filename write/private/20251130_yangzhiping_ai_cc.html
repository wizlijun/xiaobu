AI 生产力的跃迁

2025.11.25 | 讲师：阳志平

从 Chat 到 Work：两条路线的对决

AI 时代不仅是聊天的时代，而是生产力跃迁的时代。这一跃迁由两个里程碑定义：

  • 2022.11.30 ChatGPT： 网页端的对话窗口（Web Chat）
  • 2025.10.17 Claude 生态： 本地化的生产力闭环（Local Work）

核心洞察

"ChatGPT 把对话约束在网页窗口，OpenAI 走错了路（云端/GPTs）。

Claude 真正理解了生产力：本地电脑 (Local) + 系统语言 (Bash/Python) + Markdown 技能 (Skill)。这是人类科技史被严重低估的转折点。"

路径分歧：云端玩具 vs 本地工匠

OpenAI 路线

1

函数调用的陷阱

虽然发明了 Function Calling，但坚持服务器端运行。慢，且经常被打断。

2

GPTs 的死胡同

强行开新窗口/新网页。割裂了用户心流，无法触达用户本地文件。

3

训练方向跑偏

沉迷于 V3/R1 打榜，忽视了对真实工作环境（错误日志、Git历史）的训练。

2025 赢家

Claude 路线

利用本地沙盒 (UV / Bun)

利用开源社区最新的 UVX/BunX 技术，在本地创建临时、快速的虚拟环境执行 Python/JS，用完即毁。速度极快。

调用系统语言 (Shell/Bash)

AI 直接使用 `ls`, `grep`, `git`。这是电脑的原生语言，AI 变成了系统管理员。

本地上下文训练

两年来，Claude 专门针对"文件操作错误"、"代码回滚"等本地场景进行了特训。

Claude 生态的五大组件

这就是 2025 年的 AI 铁三角升级版，也是“活水插件”的基础。

MCP 协议

2024.11.25 发布

AI 界的 TCP/IP

全称 Model Context Protocol。

连接 AI 模型与本地数据/工具的标准化管道。不再需要每家公司写专门的接口，它是未来的通用协议。

Claude Code

2025.02.25 发布

本地 CLI 指挥官

简称 CC。运行在终端。

它让 AI 能够像资深程序员一样，直接操作你的文件系统，读写代码，执行 Git 命令。是人与 AI 在本地协作的界面。

Command

伴随 CC 发布

快捷指令

将复杂的提示词封装为简短的命令。

例如：输入 `/write` 自动触发"中文写作大师"提示词。极大地降低了重复输入的成本。

Hook & Agent

2025.09.29 发布

自动化与任务拆解

Hook (钩子): 满足条件 A，自动触发 B。
Sub-Agent (子代理): 自动将"帮我部署网站"拆解为 4 个子任务并行执行。这是真正的自动化。

Skill (技能)

2025.10.16 发布

Markdown 即代码

革命性创新。

不再需要写 JSON 或 Python 脚本。用 Markdown 写一段自然语言说明，它就变成了 AI 可执行的工具。让大众普及成为可能。

生产力跃迁的本质

1

从 "对话窗口" 到 "本地环境"

人类真正工作的上下文不在网页对话框里，而在本地的文件、文件夹、Git 历史中。Claude 把 AI 带到了数据源头。

2

从 "单次问答" 到 "任务闭环"

借助 Sub-Agent 和 Hook，AI 不再是聊完就忘，而是能连续执行多个任务（检查、修改、提交、部署），实现初级自主决策。

3

从 "程序员专用" 到 "大众普及"

Skill (技能) 的出现，让不懂代码的用户通过写 Markdown 文档，就能定义复杂的 AI 行为。这标志着 AI 开发门槛的彻底降低。

阳老师的预判： 硅谷各大公司（OpenAI, Google）目前还在追逐多模态等热点，尚未完全理解这套"本地化+生态协议"打法的先进性。这就是我们的时间窗口。
